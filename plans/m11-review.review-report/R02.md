## Detailed Findings

### R02-01: 20 GB free threshold not derived from model size

#### Background
Phase 2 of `_wait_for_overlap_engines_offloaded` polls `nvidia-smi --query-gpu=memory.free` and continues training once `min_free_gb >= 20.0`. The 20 GB constant was chosen to ensure the 0.5B model's ~3.7 GB weights plus activations fit with margin.

#### Expected behavior / invariant
For arbitrary model sizes, the free-memory threshold should reflect the train actor's actual GPU memory footprint.

#### Finding
The threshold is hardcoded and not derived from model size, training args, or activation footprint.

#### Evidence
- Code: `miles_pipeline.py:494` — `target_free_gb = 20.0` (literal constant)
- M11.1 attempt 5 log shows "min=28.20 GB free" and successful wake_up for 0.5B
- M11.2 smoke evidence: Disjoint pools, threshold never actually constrains training

#### Impact
Medium severity CORRECTNESS gap: Users running large models could face OOM if the driver releases memory asynchronously in parallel with train wake_up. The hardcoded 20 GB is not wrong for M11.1 but is not principled.

#### Recommended action
1. Document the 20 GB threshold in a code comment.
2. (Future) Derive threshold from `miles_args.model_size`.
3. Consider adding a `--min-free-gpu-mem-gb` flag.

#### Routing
- Debug-log update: Document as an acceptable limitation for M11.1.
- PLAN.md revision: Not required.
- Rerun needed: No.

### R02-02: No timeout backpressure on nvidia-smi subprocess

#### Background
Phase 2 subprocess call to `nvidia-smi` has a 5.0s timeout.

#### Expected behavior / invariant
When `nvidia-smi` times out, the function should signal clearly to the caller.

#### Finding
Timeout is caught as `subprocess.SubprocessError` in a broad except clause that also catches `OSError`. The debug log is at `logger.debug` level, which means timeout events are invisible in INFO-level logging.

#### Evidence
- Code: `miles_pipeline.py:550–552`
- M11.1 smoke evidence: Timeout not exercised on responsive hardware

#### Impact
Low severity OBSERVABILITY_GAP.

#### Recommended action
1. Log the timeout event at INFO level.
2. (Optional) Add a counter/metric.

#### Routing
- Debug-log update: No update needed.
- PLAN.md revision: Not required.
- Rerun needed: No.
