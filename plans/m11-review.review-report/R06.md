## Detailed Findings

### R06-F1: Unnamed lifetime="detached" coordinator may persist after driver crash

#### Background
Both drivers create coordinators with `lifetime="detached"` (lines 203 / 207). Survives driver exit.

#### Expected behavior / invariant
On driver crash/restart, either:
- Orchestrator cleans up stale coordinators
- New driver detects conflict and fails fast
- Naming scheme prevents collisions

#### Finding
`allocate_pipeline_id` returns fresh UUID-based pipeline_id on restart (orchestrator generates `f"{pipeline_type}_{uuid.uuid4().hex[:12]}"` and checks for collision). Each run gets a new pipeline_id, new namespace. Old coordinator orphaned but unreachable. Collision probability negligible (16 hex = 2^64). Even if collision, scheduler's namespace-qualified cache resolves correctly.

#### Evidence
- Code: orchestrator.py allocate_pipeline_id implementation
- types.py: COORDINATOR_ACTOR_NAME_PREFIX, get_pipeline_namespace
- M11.1 attempt 10 + M11.2 attempt 4 PASS show clean init on fresh runs

#### Impact
None on correctness. Resource observability issue (orphaned actors accumulate on long-lived clusters).

#### Recommended action
1. Document accumulation trade-off in driver docstring.
2. M11.3: add `orchestrator.cleanup_stale_pipelines()` RPC.

#### Routing
- Debug-log update: None (not a bug)
- PLAN.md revision: None (out of scope; hard constraint)
- Rerun needed: No

## Reviewer Notes

**Shard scope strictly LIMITED to coordinator naming + namespace + scheduler RPC lookup.** M11.1 attempt 4's "Failed to resolve actor" failure and fix (commit 7b83be5) is the primary evidence that this convention is load-bearing. Subsequent attempt 10 (full E2E pass) and M11.2 (dual-pipeline pass) prove the convention is stable.

**Notes on namespace hierarchy:** Ray doesn't enforce parent-child namespaces; they are flat scopes with explicit lookup. The terms "parent" and "sibling" are logical, not Ray-enforced. Scheduler in `RLIX_NAMESPACE="rlix"` resolves coordinators in per-pipeline namespaces via explicit namespace parameter. This is standard Ray practice.

**Notes on detached lifetime:** `lifetime="detached"` is correct for production (pipeline survives driver restart). Stale orphan accumulation is observability gap, out of scope per hard constraint. Cleanup is M11.3+ task.

**Cross-shard coordination:** Independent of R01-R05. R03 (Phase A/B) depends on coordinator construction succeeding (R06 proves). R04, R05 depend on coordinator namespace correctness (R06 validates).
