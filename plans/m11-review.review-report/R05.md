## Detailed Findings

### R05-01: Zombie MilesCoordinator on driver crash

#### Background
The dual driver creates two MilesCoordinator actors with `lifetime="detached"`. They persist after driver process exit.

#### Expected behavior / invariant
Either both pipelines complete, or on failure, all actors are cleaned up. The driver's docstring (lines 32–34) explicitly states "MUST NOT have top-level try/except and MUST NOT call ray.shutdown()" per F13 scope.

#### Finding
The dual driver lacks defensive cleanup on partial failure. P1's detached coordinator survives, consuming Ray cluster resources.

#### Evidence
- Code: run_miles_dual.py lines 299–322 (sequential _build_pipeline calls, no try/except)
- Code: run_miles_dual.py lines 202–207 (lifetime="detached")
- Plan: F13 scope forbids try/except per design
- Log: m11-2-dual-pipeline-log.md attempt 4 shows clean PASS

#### Impact
NOTE — M11.2 scope assumes user discipline (clean Ray cluster between runs).

#### Recommended action
1. Document P2 failure recovery in PLAN.md "Known Limitations" section.
2. Instruct users: "If a pipeline init fails, run `ray stop` to clean up zombies."
3. M11.3: optional cleanup logic.

#### Routing
- PLAN.md revision: Add "M11.2 Failure Semantics" section
- Debug-log: Record as F13-DESIGN-CHOICE
- Rerun: No

### R05-02: _split_pools_for_dual does not validate odd GPU counts

#### Background
The function validates `num_gpus_per_node >= 2 * infer_pool_size` but not modulo.

#### Finding
Accepts odd GPU counts silently. For 5-GPU machine: produces [0,1] and [2,3], leaves [4] unused.

#### Evidence
Code: run_miles_dual.py:54–72; M11.2 attempt 4 used 4-GPU machine.

#### Impact
NOTE — M11.2 targets 4-GPU machines.

#### Recommended action
1. Add explicit validation with warning or rejection.
2. Document the design choice.

#### Routing
- PLAN.md revision: Implementation note for M11.3
- Debug-log: F13-EDGE-CASE
- Rerun: No

## Reviewer Notes

1. **_split_pools_for_dual is correct for the M11.2 target case** (4-GPU, infer_pool=2).
2. **Port collision window is narrow but real** (R05-Q4). 1000-port offset + random offset prevents collisions in practice.
3. **Detached coordinator lifetime is a conscious design choice** (F13 scope). Cleanup is user responsibility via `ray stop`.
4. **No cross-pipeline shared state detected** in asyncio.gather train loops. Each Ray actor isolated.
5. **_per_pipeline_args deep-copy is correct** (line 84). Each pipeline gets fresh args.
6. **Scheduler accepts cluster_device_mappings without overlap validation** (R05-Q6). Driver guarantees disjoint via split.
7. **M11.2 PASS is credible** (attempt 4, both pipelines trained 2 rollouts, EXIT_CODE=0). Non-fatal KeyError engine_index 2 (cross-shard with R02) addressed by 549cfbd.
